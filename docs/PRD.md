# 任務管理系統 PRD (Product Requirements Document)

## 1. 專案概述

### 背景與動機
現代人生活與工作繁忙，常需要處理多項瑣碎或重要的代辦事項。為了避免重要事項被遺忘，並提升時間管理與工作效率，因此需要一個直覺且簡單易用的任務管理系統來記錄與追蹤日常任務。

### 目標用戶
- 學生：用來記錄作業死線、考試日期、報告進度等。
- 上班族：管理工作項目、會議提醒與專案進度。
- 一般個人：記錄生活瑣事（如購物清單、繳費等）。

### 核心價值主張
提供一個輕量、高效率且易於操作的待辦事項管理平台，幫助用戶輕鬆掌握任務進度，實現無壓力的日程管理。

## 2. 功能需求

以下為系統的主要功能以及對應的使用者故事 (User Story)：

1. **新增任務**
   - 作為 **使用者**，我希望 **能夠快速新增一筆任務內容與預期時間**，以便 **記錄需要完成的事項**。

2. **標記任務（例如：標記為已完成）**
   - 作為 **使用者**，我希望 **能夠將已處理完畢的任務標記狀態**，以便 **清楚辨識哪些事情已經做完**。

3. **刪除任務**
   - 作為 **使用者**，我希望 **能夠刪除錯誤輸入或已不再需要的任務**，以便 **維持清單的整潔與正確性**。

4. **顯示所有任務清單**
   - 作為 **使用者**，我希望 **能夠一覽所有正在進行與已完成的任務**，以便 **盤點目前的所有代辦事項**。

5. **依完成狀態篩選**
   - 作為 **使用者**，我希望 **能夠只查看「未完成」或「已完成」的任務**，以便 **能更專注於當前需要處理的工作上**。

## 3. 非功能需求

### 技術限制
- 前端：HTML、CSS (Vanilla CSS)、JavaScript。
- 後端：Python Web 框架 **Flask**，並配合 **Jinja2** 進行樣板渲染。
- 資料庫：使用輕量級的 **SQLite** 作為關聯式資料庫。

### 效能與安全考量
- 效能：頁面載入迅速，配合 SQLite 輕量化確保操作順暢。
- 安全：防止 SQL 注入攻擊與 XSS 攻擊。
- 介面設計：將導入現代化的介面設計（例如微動效與清楚明瞭的版面配置），提供極佳的用戶體驗。

## 4. MVP 範圍

### Must Have (必要功能)
- 新增任務（內容輸入）
- 顯示所有任務清單
- 標記任務狀態
- 刪除任務

### Should Have (重要功能)
- 依完成狀態篩選任務
- 使用者介面美化與回饋動畫

### Nice to Have (加分功能)
- 編輯任務內容
- 任務分類標籤
- 深色模式支援
- 優先順序設定
