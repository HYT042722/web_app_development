# PRD: 待辦事項管理系統 (To-Do List System)

## 1. 專案概述
### 背景與動機
在日常生活中，人們經常需要處理多項任務，容易忘記重要事項或難以安排優先順序。因此，需要一個簡單且直觀的待辦事項管理系統，幫助使用者輕鬆記錄和追蹤日常任務，提升工作與生活效率。

### 目標用戶
任何需要管理個人任務、工作項目或日常雜事的一般大眾，如學生、上班族、或自由工作者等。

### 核心價值主張
提供極簡、易用的任務管理介面，讓使用者能快速專注於「該做什麼」，而不會被複雜的操作流程干擾。

## 2. 功能需求

以下為系統核心的主要功能：
1. 新增任務
2. 標記任務完成
3. 刪除任務
4. 顯示所有任務清單
5. 依完成狀態篩選

### 使用者故事 (User Stories)
- **新增任務**：作為一名使用者，我希望「能夠快速輸入並新增任務」，以便「記錄下我需要完成的事情」。
- **標記任務完成**：作為一名使用者，我希望「能將已完成的任務打勾標記」，以便「清楚知道還有哪些事情沒做完，並獲得成就感」。
- **刪除任務**：作為一名使用者，我希望「能夠刪除不再需要或錯誤新增的任務」，以便「保持清單的整潔與最新狀態」。
- **顯示所有任務清單**：作為一名使用者，我希望「能夠瀏覽我新增的所有任務」，以便「隨時檢視我所有的待辦事項」。
- **依完成狀態篩選**：作為一名使用者，我希望「能夠切換僅查看未完成的任務或已完成的任務」，以便「專注於當下需要處理的事項」。

## 3. 非功能需求

### 技術限制
- 後端框架：使用 Python + Flask
- 前端網頁渲染：Jinja2 Template + HTML/CSS
- 資料庫系統：SQLite

### 效能與安全考量
- 效能：頁面載入時間與各項操作 (新增、刪除、修改) 的回應時間應在 1 秒以內，以提供順暢的體驗。
- 網站安全：需針對使用者輸入過濾防範 XSS (Cross-Site Scripting) 攻擊，以及防止 SQL Injection（需使用 ORM 或參數化查詢機制來保護）。

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須具備)
- 新增任務
- 標記任務完成 (切換狀態)
- 刪除任務
- 顯示所有任務清單

### Should Have (應該具備)
- 依完成狀態篩選 (全部/已完成/未完成)

### Nice to Have (加分項目)
- 任務截止日期的設定與提醒
- 任務分類（例如工作、生活、個人等標籤系統）
- 拖曳排序功能
